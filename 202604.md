PowerRenameは、Microsoftがパワーユーザー向けに開発した生産性向上ツール集「PowerToys」に含まれる機能の一つである。

# SpatialLMとは
SpatialLMは、部屋の3D点群データを読み取り、壁・ドア・窓・家具などを「構造化データ」として抽出する3D LLMである。  
普通の画像認識が「写真の中に椅子があります」と言うのに近いのに対して、SpatialLMは「この位置・向き・大きさに椅子の3Dバウンディングボックスがある」「この線分が壁で、この位置にドアがある」と言う形で、空間を機械が扱いやすいデータに変換する。  
```
入力:
  部屋の点群データ .ply

出力:
  wall_0 = 壁の位置・長さ・向き
  door_0 = ドアの位置・サイズ
  window_0 = 窓の位置・サイズ
  bed_0 = ベッドの3Dバウンディングボックス
  chair_0 = 椅子の3Dバウンディングボックス
```

このため、単なる3Dスキャンではなく、ロボットナビゲーション、AR、室内デジタルツイン、建築・インテリア解析などに使いやすい形になる。  

# 処理の流れ
```
スマホ動画 / RGB-D / LiDAR
        ↓
SLAMやスキャンアプリで点群を作る
        ↓
PLYなどの点群ファイル
        ↓
```

# 内部では？
1. 点群エンコーダ
点群はそのままLLMに入れられないので、3D空間上の点の集合をコンパクトな特徴量に変換

2. LLM
変換された3D特徴量をもとに、壁・窓・家具などを表す「シーンコード」をテキストとして生成  
論文では、SpatialLMはタスク専用ネットワークではなく、標準的なマルチモーダルLLM構成を採用し、オープンソースLLMからファインチューニングされているとされている。

# 学習データ
SpatialLMは大規模な合成室内データセットで学習されている。論文では、12,328の室内シーン、54,778部屋の点群と3Dアノテーションを含むデータセットを集めたと説明されています。Hugging Face上のSpatialLM-Datasetにも同じ説明があり、データセット全体は536GB、216,547行規模として公開されています。
この「合成だけど高品質な室内データ」を大量に使っている点が、SpatialLMの性能の土台です。ただし、実世界のスキャンはノイズ、欠損、傾き、スケールずれがあるため、公式サンプルほど安定しない場合がある。
