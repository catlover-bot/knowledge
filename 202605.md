<details><summary>Boxer:Open-world 2D検出を3D Bounding Boxへ持ち上げる手法</summary>

Boxerは、画像上の2D物体検出結果を、3D空間中の3D oriented bounding boxに変換するための手法である。入力として、posed images、カメラ内部パラメータ、重力方向、必要に応じてsparse/dense depthやsemi-dence point cloudを利用し、屋内シーン内の物体を3D空間で定位する。特に、open-vocabulary 2D detectorによって得られた検出結果を3Dへ持ち上げる点が特徴である。  

Boxerの処理は大きく3段階で構成される。まずOWLv2などのopen-vocabulary 2D detectorにより、テキストプロンプトや手動の2D bounding boxを用いて画像上の物体を検出する。次に、BoxerNetが各2D bounding boxを入力として、カメラ情報・重力方向・深度情報などを考慮し、3D oriented bounding boxを予測する。最後に、複数フレームにまたがる3D検出結果をfusionまたはtrackingによって結合し、グローバルに一貫したシーンレベルの3D物体検出結果を得る。  

技術的には、BoxerNetは画像cropをDINOv3で符号化し、カメラ・深度特徴とcross-attentionさせて3D bounding boxを推定する。既存の2D検出器を利用することで、3D検出そのものを一から学習するのではなく、「2D検出結果を3Dへliftする」問題に焦点を絞っている。これにより、高コストな3D bounding boxアノテーションへの依存を相対的に抑えつつ、open-worldなカテゴリに対応しやすくしている。  

論文ではBoxerNetはCuTRを拡張し、aleatoric uncertaintyを用いた頑健な回帰、sparse depth入力に対応するmedian depth patch encoding、大規模な3D boundingboxデータによる学習を導入している。評価ではegocentric settingにおいてdense depthなしの場合にCuTRを大きく上回り、CA-1Mでもdense depth利用時に既存手法を上回る性能が報告されている。  

Githubでは推論用コードと事前学習済みモデルが公開されており、Project Aria、CA-1M、SUN-RGBD、ScanNetなど複数のデータソースに対応している。ただし、リポジトリにはtraining codeは含まれておらず、主にinference用の実装として公開されている。  

# 技術的には  
Boxerの面白さは、2Dのopen-vocabulary検出器と3D空間理解を分離している点にある。従来の3D object detectionでは、カテゴリごとの3Dアノテーションや深度付きデータセットに強く依存しがちだが、Boxerはまず協力な2D detectorに物体候補を出させ、その後に3D liftingを行う。この設計により、2D側の語彙拡張性を活かしながら、3D空間中での物体位置・姿勢・サイズを推定できる。  
また、multi-view fusionやonline trackingによって、単一フレームごとの不安定な3D推定を、シーン全体で一貫した3D物体表現へ統合している点も重要である。これは、AR/VR、ロボティクス、屋内シーン理解、3D scene grath構築などに応用しやすい。

<details><summary>株価チャートを文章のように読ませる訓勇AIモデル</summary>

Kronosは、金融市場のデータを予測・生成するための基盤モデルである。  
普通のLLMは、文章を  
```text
私→は→学校→に→行く
```
のような**単語の並びとして学習する**。  
一方でKronosは、金融市場のローソク足データを  
```text
価格が上がった→出来高が増えた→高値を更新した→下落した
```
のような**市場の動きの並び**として学習する。  

つまり、Kronosは金融チャートを「言語」のように扱うモデルである。

# 何を入力するのか
Kronosが扱う主な入力は、**ローソク足データ**である。  
| 用語     | 意味         |
| ------ | ---------- |
| Open   | その時間の最初の価格 |
| High   | その時間内の最高価格 |
| Low    | その時間内の最低価格 |
| Close  | その時間の最後の価格 |
| Volume | その時間内の取引量  |

これらをまとめてOHLCVと呼ぶ。Kronosの実装では、open、high、low、closeが必須で、volumeとamountは任意入力として扱われている。  

# Kronosは何をしているのか
Kronosの処理は、かなり単純化すると次の流れである。  
```text
過去のローソク足データ  
    ↓  
数値をトークンに変換  
    ↓  
Transformerで市場のパターンを学習
    ↓
未来の価格・ボラティリティなどを予測
```
ここで重要なのが**トークン化**である。  
トークン化とは、データをAIが扱いやすい小さな単位に分けることである。  
文章なら「単語」や「文字」がトークンになる。  

Kronosでは、価格や出来高のような連続的な数値を、専用のtokenizerによって市場の動きを表すトークン列に変換する。論文では、Kronosは連続的な市場情報をtoken sequenceに離散化するspecialized tokenizerを導入していると説明されている。
